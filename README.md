## 1. 네트워크의 기초와 계층 구조

### 네트워크란?
네트워크(Network)란 여러 컴퓨터 및 장치들이 서로 통신하고 자원을 공유하기 위해 연결된 통로를 의미한다.

**Internet**은 전 세계의 네트워크가 상호 연결된 거대한 공용 통신망으로, 특정 중앙 통제 기관 없이 여러 ISP(통신사)들의 연결을 통해 운영된다.

### 네트워크 규모에 따른 분류
- **LAN (Local Area Network)**  
  근거리 범위(가정, 학교, 사무실)에서 사용하는 고속 통신망

- **MAN (Metropolitan Area Network)**  
  도시 단위의 중간 규모 네트워크

- **WAN (Wide Area Network)**  
  국가 또는 대륙 단위로 연결되는 광범위한 네트워크

### 데이터 전송 방식
- **Unicast**: 1:1 전송 (특정 대상에게만 데이터 전달)
- **Broadcast**: 1:All 전송 (동일 네트워크 내 모든 장치)
- **Multicast**: 1:Group 전송 (특정 그룹에만 전달)

## 2. 네트워크 참조 모델 (OSI 7 Layer & TCP/IP)

네트워크 통신은 기능별로 분리된 **계층 구조**를 통해 이루어지며, 각 계층은 상위 계층의 데이터를 받아 Header를 추가한 뒤 하위 계층으로 전달한다.

### OSI 7 계층
| 계층 | 이름 | 역할 |
|---|---|---|
| 7 | 응용 계층 | 사용자 인터페이스 제공 (HTTP, FTP 등) |
| 6 | 표현 계층 | 데이터 형식 변환, 암호화, 압축 |
| 5 | 세션 게층 | 송신자와 수신자 간 세션(논리적 연결)을 설정, 유지, 종료하는 기능을 담당 |
| 4 | 전송 계층| 신뢰성 있는 전송, 흐름/혼잡 제어 (TCP/UDP) |
| 3 | 네트워크 계층 | 다른 네트워크 간의 데이터 전달을 위한 경로를 선택하고 IP 주소 기반 라우팅을 수행 |
| 2 | 데이터링크 계층 | 프레임 형태의 로컬 네트워크 통신을 담당하고, 오류 검출/ 수정 및 충돌 제어 수행 |
| 1 | 물리 계층 | 실제 전송 매체(케이블, NIC)를 통한 비트(bit)를 전송하는 계층|

### TCP/IP 4 계층
| TCP/IP 계층 || 설명 |
|---|---|---|
| 응용 계층(Application)| 4 | IOS의 응용, 표현, 세션 계층을 하나로 통합. 사용자 인터페이스와 밀접한 서비스 제공 (HTTP, DNS, SMTP등) |
| 전송 계층(Transport)| 3 | 신뢰성 있는 데이터 전송과 프름제어를 담당. 주요 프로토콜 TCP, UDP |
| 인터넷 계층(Internet)| 2 | OSI의 네트워크 계층에 해당하며 IP 기반 라우팅 기능 수행 (IP, ICMP, ARP) |
| 네트워크 인터페아스 계층(Link) | 1 | OSI의 물리 계층 + 데이터 링크 계층을 합친 계층, 실제 매체를 통해 비트를 전송하고 프레임을 처리 |

## 3. 핵심 프로토콜 및 주소 체계

### TCP와 IP

#### TCP (Transmission Control Protocol)
- 신뢰성 있는 통신 보장
- 순서 보장, 오류 검출 및 재전송

**3-Way Handshake**
1. SYN  
2. SYN + ACK  
3. ACK  

→ 통신 전 상호 연결 상태를 확립

#### IP (Internet Protocol)
- 논리적 주소 지정
- 목적지까지의 경로(Routing) 결정

### IP 주소 vs MAC 주소
| 구분 | IP 주소 (Logical) | MAC 주소 (Physical) |
|---|---|---|
| 비유 | 현재 거주 중인 반/번호 | 고유한 주민등록번호 |
| 특징 | 변경 가능, 네트워크 위치 식별 | 장치 고유값, LAN 내 식별 |
| 길이 | 32비트 (IPv4 기준) | 48비트 |

### ARP (Address Resolution Protocol)
IP 주소를 기반으로 실제 통신에 필요한 **MAC 주소를 조회**하는 프로토콜

### Subnet Mask
IP 주소에서 **네트워크 영역과 호스트 영역을 구분**하기 위한 기준값

## 4. 네트워크 보안 및 위협 기법

### 주요 위협 요소
- **외부 위협**: 해커, APT, 국가 지원 공격 그룹
- **내부 위협**: 내부 직원의 실수 또는 악의적 행위

**보안의 3요소 (CIA)**
- 기밀성(Confidentiality)
- 무결성(Integrity)
- 가용성(Availability)

### 대표적 공격 기법
- **MITM (Man-In-The-Middle)**  
  통신 중간에 개입하여 데이터 탈취 또는 변조

- **DDoS (Distributed Denial of Service)**  
  다수의 감염된 장비로 트래픽을 유발해 서비스 마비

- **Spoofing**  
  IP 또는 MAC 주소를 위조하여 신뢰 장치로 위장

## 5. 클라우드 컴퓨팅과 보안

### 클라우드 서비스 모델
| 모델 | 설명 | 보안 책임 |
|---|---|---|
| IaaS | 인프라만 제공 | 사용자 책임 가장 큼 |
| PaaS | 개발 환경 제공 | 인프라: 제공자 / 앱: 사용자 |
| SaaS | 완성된 소프트웨어 | 계정·데이터 관리 위주 |

### 클라우드 보안 핵심
- **Zero Trust**  
  “아무도 신뢰하지 않는다”는 원칙 하에 모든 접근에 대해 지속적 인증 및 검증 수행

- **주요 취약점**  
  설정 오류(S3 공개), 권한 오남용, API 보안 미흡
